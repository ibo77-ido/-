# 2026-06-10 工作日志

## Phase6 × Phase3 接口桥接实现

### 任务
将 Phase6 的 4 个区域（Order/Wheel/Glaze/Kiln）与 Phase3 的 4 个功能模块（Order/Shape/Glaze/Firing）通过接口连接，实现"到区域按 E 键→响应对应模块→完成后等待移动到下一区域"的分步工作流。

### 修改文件

#### `Assets/Phase3/Scripts/Core/GameManager.cs`
- 新增 `IsPhase6Mode` 属性 + `EnablePhase6Mode()` / `DisablePhase6Mode()` 方法
- 新增 `public event System.Action<GameState> OnStageCompleted` 事件
- 新增 4 个 Notify 方法：`NotifyOrderCompleted()` / `NotifyShapeCompleted()` / `NotifyGlazeCompleted()` / `NotifyFiringCompleted()`
  - 正常模式下直接调用原有 `GoToXxx()`
  - Phase6 模式下触发 `OnStageCompleted` 事件（Firing 除外，Firing 完成后直接 `GoToResult`）
- `StopGameplayLoop()` 自动调用 `DisablePhase6Mode()`

#### `Assets/Phase3/Scripts/UI/OrderPanelController.cs`
- `OnAcceptClicked()` 改为调用 `gameManager.NotifyOrderCompleted()`（而非直接 `GoToShape()`）

#### `Assets/Phase3/Scripts/UI/ShapePanelController.cs`
- `OnToGlazeClicked()` 改为调用 `gameManager.NotifyShapeCompleted()`

#### `Assets/Phase3/Scripts/UI/GlazePanelController.cs`
- `OnToFiringClicked()` 改为调用 `gameManager.NotifyGlazeCompleted()`

#### `Assets/Phase3/Scripts/UI/FiringPanelController.cs`
- `OnOpenKilnButtonClicked()` 改为调用 `gameManager.NotifyFiringCompleted()`（内部直接 GoToResult）

#### `Assets/Phase6/Scripts/Systems/TestUIRouter.cs`（重写）
- 新增 `phase6Progress` 整型进度追踪（0=未开始 1=Order完成 2=Shape完成 3=Glaze完成 4=结束）
- `OpenGameplayUI()` 根据区域类型和进度决定是否允许进入，并调用对应 Phase3 方法
- `CanOpenGameplayArea()` + `GetAreaUnlockHint()` 实现顺序保护
- `CloseGameplayUIWithoutReset()` 阶段完成后只关 UI 不 StopGameplayLoop（保留数据状态）
- `OnPhase3StageCompleted()` 订阅 GameManager.OnStageCompleted，更新进度并关 UI
- `NotifyGameplayCompleted()` 游戏完全结束时重置进度

### 核心架构决策
- **不修改 Phase3 的 GoToXxx 方法**，保持 Phase3 场景完全兼容，只在 Notify 层做分叉
- Phase6 模式下的 "挂起" 通过 `CloseGameplayUIWithoutReset()` 实现——关闭 UI 但不调用 StopGameplayLoop，GameManager 数据状态保留
- Kiln（烧制）区域完成后直接 GoToResult，不需要挂起等待，因为 Result 是最终阶段
