## 二、FiringActive 态（烧制主态）

### 2.1 主画面视频（两段式）

```
EnterState(FiringActive, force:true)
  → staticFallbackImage.SetActive(false)         // 隐藏底图，让视频可见
  → InitTemperatureVideo()                        // 温度条归零 + Prepare
  → PlayFireStartSequence()
     → firingLoopPlayer.Stop()
     → fireStartPlayer.frame = 0
     → fireStartPlayer.loopPointReached += OnFireStartFinished
     → fireStartPlayer.Play()                     // 播放 火焰动画.mp4（起势，单次）

fireStartPlayer.loopPointReached
  → OnFireStartFinished()
     → fireStartPlayer.Stop()
     → firingLoopPlayer.frame = 0
     → firingLoopPlayer.Play()                    // 播放 循环.mp4（循环）
```

- 两个 VideoPlayer 共享 RT_Firing RenderTexture
- RawImage `FiringVideo_RawImage` 显示输出
- 起势视频单次播放，循环视频 isLooping=true

### 2.2 玩家操作

| 操作 | 输入 | 系统响应 |
|------|------|---------|
| 添柴 | 点击 Btn_Fuel | `firingSystem.AddFuel()` → currentTemperature += 200 |
| 调风门 | 鼠标滚轮在 WindDoor_HoverArea | windValue ±1/8（9档），调 `SetWindValue()` |
| 停火 | 点击 Btn_Stop | `StopFiring()` + `EnterState(ClosingKiln)` |

风门交互细节（`UpdateWindDoorScroll`）：
- 只在 FiringActive 态响应
- `RectTransformUtility.RectangleContainsScreenPoint` 检测鼠标在热区（Overlay 模式 camera=null）
- 滚轮向上 → windValue += 0.125；向下 → -= 0.125
- Clamp 0~1，调 `firingSystem.SetWindValue()`
- `RefreshWindDoorFrame` 按帧索引切换 sprite（带缓存）：
  - windValue=0 → 1.png（第0帧，最小开度）
  - windValue=1 → 满.png（第8帧，最大开度）

### 2.3 温度数值计算（FiringSystem.Update 每帧）

```
windValue >= 0.3:
    lowWindTimer = 0
    isTemperatureDropping = false
    currentTemperature += 50 × windValue × deltaTime

windValue < 0.3:
    lowWindTimer += deltaTime
    持续 < 3s: 仍按升温算
    持续 >= 3s:
        isTemperatureDropping = true
        currentTemperature -= 30 × deltaTime（不低于0）
```

### 2.4 温度条视频（独立于温度数值）

Controller 内部维护 `temperatureBarProgress`（0~1），与 `CurrentTemperature` 完全解耦。

```
UpdateTemperatureVideo()（每帧）
  → 非 FiringActive 态 / hasTriggeredForceOpen / 未 Prepared → return
  → IsTemperatureDropping == true:
       isReversing = true
       temperatureBarProgress -= temperatureBarReverseSpeed × deltaTime
  → IsTemperatureDropping == false:
       isReversing = false
       temperatureBarProgress += temperatureBarForwardSpeed × deltaTime
  → targetFrame = (int)(progress × totalFrames)
  → targetFrame != lastTemperatureBarFrame → videoPlayer.frame = targetFrame
  → isReversing && progress <= 0 → TriggerForceUnderfiredOpen()
```

- 温度条视频是风门状态的视觉趋势反馈，不与 CurrentTemperature 一一对应
- CurrentTemperature 仅用于评分
- windValue 恢复 >= 0.3 → IsTemperatureDropping 立即 false → 倒放停止，恢复正向（第一版无 decay window）

### 2.5 UI 元素表现

| 元素 | 表现 |
|------|------|
| 主画面 | 火焰动画.mp4 → 循环.mp4（铺满主体） |
| 温度条 | 温度条变化.mp4 按帧 seek（960x120，底部居中） |
| 风门 | 9 帧序列图按 windValue 切换（右侧中下，200x300） |
| 按钮 | Btn_Fuel/Btn_Stop active；Btn_OpenKiln inactive |
| 旧控件 | windSlider/windowButton/Text_* 全部 inactive |

## 三、ClosingKiln 态（停火过渡）

### 触发


### EnterState(ClosingKiln) 行为
```
  → fuelButtonRoot.SetActive(false)
  → stopButtonRoot.SetActive(false)
  → openButtonRoot.SetActive(false)

```

```
  → OnClosingKilnFinished()

```

## 五、强制开窑分支（次品）

### 触发
温度条视频倒放到第 0 帧（风门低进风持续 3s 导致）

### 流程
```
UpdateTemperatureVideo()
  → isReversing && temperatureBarProgress <= 0 && !hasTriggeredForceOpen
  → TriggerForceUnderfiredOpen()

TriggerForceUnderfiredOpen()
  → hasTriggeredForceOpen = true                 // 防重复
  → firingSystem.ForceUnderfiredOpen()
     → currentTemperature = 0
     → isTemperatureDropping = false
     → lowWindTimer = 0
     → StopFiring()                               // isFiring=false, isWindowOpen=true
  → StopMainVideo()                               // 停主画面视频，显示底图
  → closingKilnPlayer.Stop()                      // 跳过 ClosingKiln 动画
  → gameManager.GoToResult()                      // 直接进入结果
```

### 次品判定原理
- `ForceUnderfiredOpen()` 明确把 `currentTemperature = 0`
- `CalculateScore()` 的 `stopFireTemp = 0`
- `GetFireScore()` 返回 0
- `GetCurrentZone()` 返回 `Underfired`
- ResultSystem 按现有逻辑判次品，**不改 ResultSystem**

## 九、各组件职责

| 层 | 组件 | 职责 | 不越界 |
|----|------|------|--------|
| 数值层 | FiringSystem | 算温度、判区间、算分数、ForceUnderfiredOpen | 不碰视频/UI |
| 表现层 | FiringPanelController | 视频播放、风门切帧、按钮显隐、状态切换、温度条 seek | 只读 IsTemperatureDropping/WindValue，不改数值 |
| 评分层 | ResultSystem | 按现有逻辑判次品 | 不改，靠 stopFireTemp=0 → 分数=0 天然次品 |
| 构建层 | Phase3SceneBuilder | ArtRoot_Firing 存在时跳过旧占位控件 | 不影响其他面板 |

### FiringPanelController 状态入口

```csharp
EnterState(FiringUiState newState, bool force = false)
```
- 唯一状态入口，负责一次性显隐、视频播放/停止、按钮启用
- `force=true` 用于 ResetPanel 强制重置

### Refresh 职责分离

- **EnterState**：一次性显隐、视频播放/停止、按钮启用
- **Refresh**：连续刷新（文本/IsFiring 检测），不做状态切换
- **UpdateWindDoorScroll / RefreshWindDoorFrame**：风门滚轮输入 + 序列帧切换
- **UpdateTemperatureVideo**：温度条视频帧驱动 + 倒放 + 强制开窑触发

## 十、数据与表现分离原则

| 数据 | 用途 | 驱动者 |
|------|------|--------|
| CurrentTemperature | 评分 | FiringSystem.Update（升温/降温） |
| IsTemperatureDropping | 决定温度条视频方向 | FiringSystem（windValue<0.3 持续3s） |
| WindValue | 风门帧映射 + 温度计算 | 玩家滚轮 → SetWindValue |
| temperatureBarProgress | 温度条视频帧 seek | FiringPanelController（独立于温度数值） |

- `temperatureBarProgress` 是 Controller 内部 0~1 进度，与 `CurrentTemperature` 完全解耦
- 温度条视频是风门状态的视觉趋势反馈，不与温度数值一一对应
- 数值仅用于评分

## 十一、Phase3SceneBuilder 兼容策略

```
EnsureFiringPanelControls(canvasTf)
  ├─ 找 Panel_Firing
  │   ├─ 不存在 → 返回
  │   └─ 存在
  │       ├─ 检测 ArtRoot_Firing
  │       │   ├─ 存在 → 仅确保 FiringPanelController，返回（跳过占位控件）
  │       │   └─ 不存在 → 走原有逻辑（创建 Text_Zone/Slider_Wind/Btn_*）
```

- 美术面板接入后，Builder 不会创建旧占位控件污染 Panel_Firing
- ArtRoot_Firing 节点名不可重命名（Builder 检测依赖）

## 十二、已知风险与待优化项

### 已知风险（第一版不处理，观察后再加）

1. **起势视频潜在重播**：Refresh() 的 IsFiring 变更触发 EnterState(FiringActive, true) 与 ResetPanel() 可能导致起势视频轻微重播——运行时若观察到回跳再收紧进入条件
2. **余热 decay window**：IsTemperatureDropping 在 windValue 恢复时立即切换为 false，温度条视频倒放→正向会有视觉突变——若实测明显再加 Controller 表现层本地计时器
3. **温度条视频 seek 性能**：每帧 `videoPlayer.frame = targetFrame` 可能有开销——已带 lastTemperatureBarFrame 缓存，若卡顿降低 seek 频率
4. **触屏设备无滚轮**：第一版不处理

### 待运行时验证

- 起势→循环视频无缝衔接
- 停火→关窑动画→打开按钮流程
- 风门滚轮 9 档切换
- 低风门 3s 倒放→强制开窑次品
- 风门/温度条位置对照参考图精调
