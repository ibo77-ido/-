### 2. 项目定位

《继承瓷厂》是一款以中国传统陶瓷文化为核心的探索经营游戏。

玩家将继承一座濒临倒闭的小窑厂，在老窑工的引导下，从最基础的制瓷流程开始逐步成长。

项目的核心体验包括：

- 亲手参与制瓷过程
- 体验窑变带来的结果惊喜
- 在玩法中自然融入陶瓷文化表达
- 通过经营和成长建立持续目标感

目标用户主要包括：

- 文化题材玩家
- 模拟经营玩家
- 独立游戏玩家
- 收藏探索型玩家

---

### 3. 核心玩法闭环

项目的 MVP 核心循环为：

**接订单 -> 制作器型 -> 配置釉料 -> 控制火候 -> 开窑 -> 鉴赏评级 -> 获得奖励 -> 窑厂成长**

这个循环的意义在于：

- 让玩家明确知道每一步在做什么
- 让制瓷过程形成完整闭环
- 让结果有明确反馈，而不是单纯的数值堆叠
- 让“再试一次”的动力成立

项目的重点不是做一个简单的生产流程，而是让玩家在反复尝试中逐渐理解制瓷的过程与乐趣。

---

### 4. 评分系统与结算逻辑

器型评分已经从原来的五点细分，调整为三个核心区域：

- 颈部
- 躯干
- 底部

这三个区域不是独立判断，而是相互影响。

三个核心区域会通过系数关系影响原先的五个细分点，评分本质仍然是对器型匹配度的判断，只是结构更清晰、层次更合理。

评分系统的设计逻辑是：

- ShapeScore 负责器型评分
- GlazeScore 负责釉色评分
- FireScore 负责火候评分
- ResultCalculator 负责最终汇总

其中，ResultCalculator 不重新打分，只负责把三个评分结果整理成最终分数、等级、订单结果和奖励。

Phase6 的价值不只是“做出一个场景”，而是建立一套可以继续扩展的空间化基础架构。

### 7. Phase10 叙事层

Phase10 不是替代现有玩法层，而是建立叙事内容层。

Phase10 的核心结构包括：

- Narrative State Machine
- Narrative EventBus
- Narrative CommandBus
- Bridge Adapter

Phase10 的设计原则是：

- 叙事层与 Phase3、Phase6、Phase8 保持职责隔离
- 不让旧系统被剧情逻辑反向污染
- 为后续章节、角色、事件和任务扩展预留接口

Chapter 1《重燃窑火》就是 Phase10 的首个验证章节，用来测试叙事层是否能独立运行、稳定桥接并持续扩展。
