## Task Design Rule

Each task must be small enough to inspect, implement, and validate in one UnityMCP pass.

## Scope Rule

- One task should cover one decision or one visible layer step.
- Do not combine asset audit, scene wiring, and final tuning in one task.
- Keep the first pass focused on map readability, not final polish.
