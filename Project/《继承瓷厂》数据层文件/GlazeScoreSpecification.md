## 1. 评分原理

### 1.1 前提

GlazeCalculator.md 已建立基于 Cu/Fe/Co 三元素的欧几里得距离算法：

```
d_min = min( sqrt((Cu_p - Cu_i)² + (Fe_p - Fe_i)² + (Co_p - Co_i)²) )
        i ∈ {影青, 甜白, 青花, 霁红, 冬青}
```

- `d_min = 0`：玩家配方与某个标准釉色完全一致
- `d_min > 0`：配方有偏差，数值越大偏差越严重

### 1.2 评分目标

将距离 `d_min` 映射为 **0~100** 分的 GlazeScore：

| 关系 | 映射方向 |
|------|----------|
| `d_min` → 0（距离极小） | GlazeScore → 100（高分） |
| `d_min` → D_ref（参考距离） | GlazeScore → 0（零分） |
| `d_min` → ∞（距离极大） | GlazeScore → 0（下限） |

### 1.3 参考距离 D_max

D_max 定义为"配方与目标釉色之间的最大可接受偏差"。超过此偏差，配方在化学上不再属于该釉色范畴。

**D_max 取值依据：**

5种标准釉色在 Cu/Fe/Co 三维空间中的分布：

| 釉色对 | 向量距离 | 含义 |
|--------|----------|------|
| 甜白 ↔ 影青/冬青 | 0.005000 | 铁系釉色间最近距离 |
| 影青 ↔ 霁红 | 0.022361 | 铜系与铁系间距离 |
| 甜白 ↔ 霁红 | 0.018028 | 铜系与铁系间距离 |
| 青花 ↔ 霁红 | 0.022361 | 钴系与铜系间距离 |
| 甜白 ↔ 青花 | 0.025000 | 铁系与钴系间距离 |
| 平均釉间距离 | **≈ 0.018350** | |

> 计算：取 D_max = 0.02，略大于平均釉间距离，确保"完全搞错釉色"的配方得分为0或接近0。

**选择 0.02 的理由：**

- 略大于平均釉间距离(0.01835)，确保"明确偏向另一个釉色"的配方得低分
- 0.02 是 Cu/Fe/Co 单元素的最大差值（各元素取值范围 0~0.02），直观且物理意义明确
- 线性公式下 D_max=0.02 使分数在关键区间分布合理

## 2. GlazeScore 公式

### 2.1 主公式

```
d_min = min( EuclideanDistance(玩家向量, 模板向量_i) )

GlazeScore = 100 × max( 0, 1 − d_min / D_max )

其中 D_max = 0.02
```

### 2.2 公式解释

| 变量 | 含义 | 来源 |
|------|------|------|
| d_min | 玩家配方到最近标准釉色的欧几里得距离 | GlazeCalculator.md §2 |
| D_max | 参考距离（分值归零点） | 本规范 §1.3 |
| GlazeScore | 最终评分，0~100 | 本公式 |

### 2.3 数学性质

- **单调性**：d_min 严格递减时，GlazeScore 严格递增（距离越小分数越高）
- **连续性**：d_min 连续变化时 GlazeScore 连续变化，无断点
- **值域**：[0, 100]，两端闭区间
- **对称性**：Cu/Fe/Co 三元素的偏差对分数的贡献对称（等权重）
- **无负数**：d_min ≥ D_max 时自动归零

### 2.4 评分等级映射

| 分数区间 | 等级 | 名称 | 说明 |
|----------|------|------|------|
| 95 ~ 100 | **S** | 完美 | 配方与标准釉色完全一致，仅极微小波动可接受 |
| 85 ~ 94 | **A** | 优等 | 配方接近标准，±1个元素微偏 |
| 70 ~ 84 | **B** | 良好 | 配方有轻度偏差，但仍明确属于该釉色 |
| 50 ~ 69 | **C** | 及格 | 配方有明显偏差，方向正确但精度不足 |
| 30 ~ 49 | **D** | 偏差 | 配方严重偏离，接近混淆边界 |
| 0 ~ 29 | **E** | 失败 | 配方与该釉色无关，或更接近其他釉色 |

## 4. 边界案例分析

### 4.1 分值归零边界

```
GlazeScore = 0 ⟺ d_min ≥ D_max = 0.02
```

| 输入 | 最近釉色 | d_min | 是否归零 | 含义 |
|------|----------|-------|----------|------|
| Cu=0.020, Fe=0.000, Co=0.000 → 青花 | 青花 | 0.020000 | **是 | 恰好边界 |
| Cu=0.000, Fe=0.020, Co=0.020 → 影青/冬青 | 影青/冬青 | 0.020000 | **是** | 恰好边界 |
| Cu=0.010, Fe=0.015, Co=0.010 → 任何 | 甜白 | 0.018028 | 否(Score≈9.9) | 接近边界 |

### 4.2 等分边界

| 分数 | d_min 精确值 | 含义 |
|------|-------------|------|
| 100.0 | 0.000000 | 完美匹配 |
| 95.0 | 0.001000 | S/A 分界线 |
| 85.0 | 0.003000 | A/B 分界线 |
| 70.0 | 0.006000 | B/C 分界线 |
| 50.0 | 0.010000 | C/D 分界线 |
| 30.0 | 0.014000 | D/E 分界线 |
| 0.0 | 0.020000 | E 下限 |

### 4.3 釉色混淆边界

当 d_min 接近"到次近釉色的距离"时，配方处于混淆状态。

| 混淆对 | 中点距离 | 对应 GlazeScore | 含义 |
|--------|----------|-----------------|------|
| 甜白 ↔ 影青 | 0.002500 | 87.5 (A) | 铁系内混淆，仍属良好 |
| 影青 ↔ 霁红 | 0.011180 | 44.1 (D) | 跨系混淆，品质下降 |
| 青花 ↔ 霁红 | 0.011358 | 43.2 (D) | 跨系混淆，品质下降 |

> 注意：甜白↔影青中点仍在 A 级（87.5分），因为两者同属铁系，距离本身就很近（0.005）。而跨系混淆的中点落入 D 级，合理反映配方定位不清。

### 4.4 D_max 选择敏感性

若 D_max 取值不同，对评分的影响：

| 案例 | d_min | D_max=0.015 | D_max=0.020（选定） | D_max=0.025 | D_max=0.030 |
|------|-------|-------------|---------------------|-------------|-------------|
| TC-06 (微偏) | 0.002000 | 86.7 | **90.0** | 92.0 | 93.3 |
| TC-13 (边界) | 0.011358 | 24.3 | **43.2** | 54.6 | 62.1 |
| WC-01 (最大铜) | 0.010000 | 33.3 | **50.0** | 60.0 | 66.7 |

> D_max=0.02 在"区分度"和"宽容度"之间取得平衡：微偏获 A 级(90分)不过低，边界落入 D 级(43分)不过高。

## 6. 釉色分类系统（V1.1 新增）

### 6.1 分类目的

将 5 种釉色按烧成气氛归类，为未来与 FireScore 联动提供语义层：

```
6.1 分类目的

将5种釉色按烧成气氛归类，用于：
- FireScore 阶段自动判断玩家选择的烧制气氛是否正确
- 玩家提示："该釉色需要还原焰烧成"
```

### 6.2 分类定义

| 分类 | 代号 | 特征 | 釉色 |
|------|------|------|------|
| **氧化釉** | `Oxidation` | 在氧化气氛中呈色，不需要还原焰 | 影青釉、霁红釉 |
| **还原釉** | `Reduction` | 必须在还原气氛中呈色，还原不到位则发色失败 | 青花、甜白釉 |
| **双性釉** | `Dual` | 氧化或还原均可烧成，但呈色效果不同 | 冬青釉 |

### 6.3 分类规则

```
Oxidation: 影青釉(Yingqing), 霁红釉(Jihong)
  → 烧制时氧化气氛为主，还原不是必须条件

Reduction: 青花(BlueWhite), 甜白釉(SweetWhite)
  → 烧制时必须经历还原阶段(1000~1280°C)，否则发色失败

Dual: 冬青釉(Dongqing)
  → 氧化烧出偏青绿色调，还原烧出偏粉青色调，两种风格均可
```

### 6.4 与 FireScore 联动

| 场景 | 联动规则 |
|------|----------|
| 玩家选择 Reduction 釉色但未开启还原焰 | FireScore 中 CP3 和 FS2 权重加倍（核心玩法关联） |
| 玩家选择 Oxidation 釉色但误开还原焰 | 触发烟熏缺陷（D9），同时扣减 FireScore |
| 玩家选择 Dual 釉色 | 气氛选择灵活，不做强约束 |

> 此分类系统将保存在 GlazeSO 的 `GlazeColorType` 字段（枚举：`Oxidation` / `Reduction` / `Dual`），不修改数据库。

## 7. 影青/冬青二级判定（V1.1 正式化）

### 7.1 问题

影青釉与冬青釉的 Cu/Fe/Co 向量完全相同（均为 (0, 0.020, 0)）。仅凭元素配方无法区分。这是化学层面的重叠，非算法缺陷。

### 7.2 三级判定机制

```
Level 1 (GlazeScore): Cu/Fe/Co 距离判定
├── 若 d_min < 0.02 且最佳匹配非 {影青, 冬青} → 输出最佳匹配
└── 若 d_min < 0.02 且最佳匹配为 {影青, 冬青} → 进入 Level 2

Level 2 (Temperature): 烧制温度判定
├── 有烧制温度数据 且 T_max ≤ 1280°C → 输出影青釉
├── 有烧制温度数据 且 T_max > 1280°C → 输出冬青釉
└── 无烧制温度数据 → 进入 Level 3

Level 3 (Fallback): UI 展示策略
└── 标记为"影青/冬青（需烧制验证）"，让玩家手动确认
```

### 7.3 判定规则

| 条件 | 输出 | GlazeScore |
|------|------|------------|
| d_min ≠ min(影青距离, 冬青距离) | 直接输出最佳匹配釉色 | 正常计算 |
| d_min 匹配影青/冬青 + T_max ≤ 1280 | 影青釉 | 100（完美） |
| d_min 匹配影青/冬青 + T_max > 1280 | 冬青釉 | 100（完美） |
| d_min 匹配影青/冬青 + 无温度数据 | 影青/冬青(待验证) | 100（完美） |

> 此判定不影响 GlazeScore 分值（d_min=0 时始终为 100 分），仅影响展示给玩家的釉色名称。不修改数据库。

## 7. 与 FireScore 的关系

| 维度 | GlazeScore | FireScore |
|------|-----------|-----------|
| 评估对象 | 釉料配方准确性 | 烧窑工艺质量 |
| 输入 | 玩家 Cu/Fe/Co 配比 | 温度曲线、时长、火焰判断 |
| 算法 | 欧几里得距离 → 线性衰减 | 多维度扣分制 |
| 输出 | 0~100 | 0~100 |
| 时机 | 配方提交时计算 | 烧窑结束时计算 |
| 关系 | GlazeScore 回答"配方对不对" | FireScore 回答"烧得好不好" |

```
最终成品评价 = f(GlazeScore, FireScore)
               = GlazeScore × FireScore / 100

例如：
  配方非常精准（GlazeScore=95）+ 烧制完美（FireScore=90）
  → 最终 = 95 × 90 / 100 = 85.5 → A级成品
```

> 最终成品评价公式为未来 Phase 的设计方向，本阶段仅定义 GlazeScore 本身。
